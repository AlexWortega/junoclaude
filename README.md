# JunoClaude

Плагин Claude Code для игры **Juno: New Origins** (SimpleRockets 2): сборка
аппаратов, программы полёта на Vizzy, запуск и разбор испытаний.

Состоит из двух частей:

- **Плагин** — скилл и MCP-сервер, работающие напрямую с файлами игры. Работает
  сразу, ничего дополнительно ставить не нужно.
- **Мод-мост** (`mod/`) — HTTP-сервер внутри игры для живой телеметрии и
  управления полётом. Требует Unity 2022.3.62f3 для сборки; инструкция в
  `mod/README.md`.

## Установка

```bash
git clone <repo> junoclaude
cd junoclaude
npm install && npm run catalog     # каталоги деталей и блоков
cd mcp && npm install && npm run build
```

Затем подключить каталог как плагин Claude Code.

Пути игры определяются автоматически для macOS и переопределяются переменными
`JUNO_INSTALL_DIR`, `JUNO_USER_DIR`, `JUNO_LOG_PATH`.

## Инструменты

| Инструмент | Назначение |
|---|---|
| `game_state` | Установка, запущена ли игра, настройки, моды |
| `craft_list` | Список конструкций |
| `craft_read` | Сводка, дерево деталей или XML выбранных деталей |
| `part_lookup` | Типы деталей, точки крепления, рецепты соединений |
| `vizzy_read` | Программа полёта в компактном DSL |
| `vizzy_write` | Компиляция DSL в Vizzy XML и запись |
| `game_launch`, `game_quit` | Управление процессом игры |
| `log_read` | Ошибки из `Player.log` со стеками |
| `junoclaude_restore` | Откат к снимку перед записью |

## Как это устроено

Форматы игры не документированы, поэтому знание о них **добыто из файлов,
которые игра написала сама**, а не выведено из рассуждений:

- `scripts/build-parts-catalog.mjs` извлекает определения 70 типов деталей из
  `SimpleRockets2_ModTools.unitypackage` и классифицирует точки крепления;
- `scripts/mine-connections.mjs` считает частоты сочетаний точек крепления по 61
  готовой конструкции и раскладывает их по видам стыковки. Это оказалось
  необходимо: самый частый рецепт для пары `Fuselage1→Fuselage1` — вовсе не
  стековая стыковка, а крепление к боковой поверхности;
- `scripts/mine-vizzy-blocks.mjs` собирает реальные строки `style` для 45
  инструкций и 67 выражений Vizzy.

Каталоги коммитятся в репозиторий, потому что установка плагина не запускает
сборку.

## Проверка целостности

Разбор и обратная сборка дают побайтово исходный файл:

- конструкции — 62 из 62,
- подсборки — 5 из 5,
- программы полёта — 7 из 7.

То есть модель не теряет ни одного атрибута из того, что мы порождаем.
Расхождения остаются на файлах `GameState`/`FlightState` из поставки, где
встречаются пустые теги вида `<X></X>`; эти файлы правятся точечной заменой,
а не пересборкой дерева.

## Безопасность

Плагин пишет в каталог сохранений игры, поэтому:

- перед каждой записью делается снимок в `.junoclaude-backups/`, откат — одним
  вызовом `junoclaude_restore`;
- запись при запущенной игре запрещена: игра держит `UserData` в памяти и
  перезапишет файл при сохранении;
- запись атомарная — игра никогда не увидит наполовину записанный файл.
