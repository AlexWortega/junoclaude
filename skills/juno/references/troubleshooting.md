# Что пошло не так

Таблица симптомов. Второй столбец — вероятная причина, третий — как проверить.

## Конструкция

| Симптом | Причина | Проверка |
|---|---|---|
| Аппарата нет в списке в редакторе | Файл не в `UserData/CraftDesigns/`, либо игра была запущена в момент записи и перезаписала каталог | `game_state` покажет, была ли игра запущена; `craft_list` — видит ли файл плагин |
| Аппарат открылся, но деталей меньше, чем в XML | Игра отбросила детали при загрузке: недостижимы от корневой либо неизвестный `partType` | `craft_read` в режиме `summary` — раздел «Предупреждения» перечислит несвязанные детали |
| Деталей столько же, но конструкция «разваленная» | Соединения указывают неверные точки крепления | `craft_read` в режиме `xml` для пары деталей, сверить с `part_lookup` |
| Корпус выглядит сегментированным, аппарат тормозит в атмосфере | В стыковке указана только пара `load`, без пары `shell` | Для `Fuselage1→Fuselage1` должно быть `a="2,4" b="1,5"`, а не `a="2" b="1"` |
| Разваливается сразу после старта | Группировка `<Bodies>` не совпала с реальными шарнирами и отделителями | Сохранить аппарат из редактора поверх нашего и сравнить `<Bodies>` |
| Баки пустые, тяги нет | `<FuelTank capacity>` не пересчитался | Сверить с сохранённым игрой файлом: если `capacity` там иной, наша оценка неверна |
| Двигатель не запускается | Деталь на другой ступени, чем ожидалось, либо не связана с командным модулем | Раздел «Ступени» в сводке |

## Программа полёта

| Симптом | Причина | Проверка |
|---|---|---|
| Компилятор отверг операцию | Имя не из DSL | Сообщение содержит путь к месту ошибки и похожие имена; полный список — в `vizzy-blocks.md` |
| Компилятор отверг свойство аппарата | Опечатка либо свойство существует только для MFD | Список свойств в `vizzy-blocks.md` |
| Программа открылась, но блоки лежат друг на друге | Верхнеуровневым блокам не хватило `pos` | Компилятор расставляет `pos` сам; при ручной правке XML — восстановить |
| Программа не запускается в полёте | Нет обработчика `FlightStart`, либо программа в детали, не являющейся вычислителем | `vizzy_read` покажет обработчики; носителем может быть командный модуль, диск, чип или MFD |
| Программа исчезла после сохранения из игры | Программа была записана, пока игра была запущена | `game_state` перед записью; восстановить снимком |

## Общее

| Симптом | Причина | Проверка |
|---|---|---|
| Инструмент отвечает `game_running` | Игра запущена | `game_quit`, затем повторить |
| Инструмент отвечает `file_busy` | Файл изменён меньше трёх секунд назад | Подождать и повторить |
| Каталог деталей не читается | Плагин установлен без сгенерированных каталогов | `npm run catalog` в корне плагина |
| `game_state` предупреждает о расхождении версий | Игра обновилась после сборки каталога | Пересобрать каталог: форматы могли измениться |

## Чтение лога

`log_read` по умолчанию показывает только ошибки, вместе со стеком и схлопывая
повторы. Полезные фильтры:

- `filter: "errors"` — исключения игры и модов;
- `filter: "mods"` — всё, что касается загрузки модов; здесь видно, принят ли
  `.sr2-mod` и не отвергнут ли он по несовпадению версии;
- `filter: "all"` — хвост лога целиком, когда нужен контекст вокруг события.

Отсутствие ошибок в логе не означает, что аппарат исправен: отброшенные при
загрузке детали игра может не считать ошибкой. Число деталей в редакторе
сравнивайте с числом в XML.
